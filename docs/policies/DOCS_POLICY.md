# Documentation Policy (LiNKsites)

## Goal
Make documentation discoverable and current without hunting across the repo.

## Rules
1. **Repo‑level docs live in `/docs`**.
2. **App‑specific docs stay in their app** (e.g., `apps/web-master/docs`).
3. **Outdated docs are archived or removed**.
4. **README must link to `/docs` index**.

## What goes in `/docs`
- Product requirements and plans
- Operations SOPs
- Deployment/VPS guidance
- Security reports
- Repo‑wide procedures

## What stays in apps
- Component guides
- CMS mapping details
- App‑specific audits and checklists

## Change control
- Any new doc must be linked in `/docs/README.md`.
- When a doc becomes obsolete, move it to an `/archive` folder with a short note or delete it.
